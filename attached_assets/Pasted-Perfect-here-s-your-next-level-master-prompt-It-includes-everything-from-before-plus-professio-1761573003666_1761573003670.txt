🚀 🔥 Ultimate Full-Stack Prompt for Building an Advanced Daily Planner System

> Prompt:



Build a professional full-stack Daily Planner System web application using:

Frontend: HTML5, CSS3, JavaScript (ES6), Bootstrap 5, Tailwind CSS, and basic animations using AOS.js and Framer Motion-like transitions.

Backend: Python (Flask Framework)

Database: SQLite3

Authentication: Flask-Login and Werkzeug Security for secure Register/Login/Logout.



---

🎯 Core Functional Requirements

1. Home Page

Clean, responsive landing page with a brief description of the system.

Include a navigation bar with sections:

Home

Tasks & Progress

Calendar

Notes & AI Chatbot

Settings

Login/Register (if user not logged in)

Logout (if user logged in)


Smooth scrolling and fade-in animations for text and cards.



---

2. Register & Login System

Register Page:

Fields: Full Name, Email, Password, Confirm Password.

Client-side validation (JavaScript) + server-side validation (Flask).

Password hashing with Werkzeug.security.

Flash success/error messages styled with Bootstrap Toasts.

Animated entry form with glowing input focus effects (Tailwind transitions + AOS fade-in).


Login Page:

Fields: Email, Password.

Remember Me checkbox.

Authentication with Flask-Login.

Redirect to dashboard after successful login.

Show “Invalid credentials” in animated toast alerts.


Logout Route: /logout clears session and redirects to login.



---

3. Task Management Page

Add Task Form:

Fields: Task Name, Description, Date, Time, Frequency (Daily, Weekly, Monthly).

Store in SQLite database with relation to the logged-in user.


Task List:

Display all tasks with edit/delete buttons.

Use colored badges for frequency (daily-blue, weekly-green, monthly-yellow).

Animated task cards with fade-in and hover elevation.


Progress Overview:

Progress bar (percentage completion).

Below: “Completed” and “Incomplete” counters.

Reward badge animation every week (e.g., confetti or animated badge using Lottie).


Export Options:

Buttons for “Export as PDF” and “Export as CSV” at bottom.

PDF shows tasks in a styled table format.


Notifications:

JS Notifications API for reminders.

Optional alarm sound when a task is due.




---

4. Calendar Page

Use FullCalendar.js for an interactive visual calendar.

Shows all tasks with their due dates.

Click any date → opens a modal to add a new task for that day.

Right-bottom corner: “Export Calendar as PDF” button.

Smooth transition animations when switching months.



---

5. Notes & AI Chatbot Page

Notes section:

Create, Edit, Delete notes stored in SQLite3.

Rich-text editor (like Quill.js).

Animated transitions when adding/deleting notes.


AI Chatbot section (placeholder):

Responsive panel design with message bubbles.

Placeholder for later AI integration (Flask route /chatbot).




---

6. Settings Page

Theme Customization:

Toggle between Light and Dark mode.

Store theme preference in localStorage.


Layout Options:

Switch between Grid/List views for tasks.

Compact/Expanded view toggle.


Personalized View:

Users can choose Day/Week/Month view.

Highlight overdue or high-priority tasks with red borders.


Smooth slide transitions between sections.



---

💾 Backend (Flask + SQLite3)

Flask routes:

/ → home
/register → register new user
/login → login existing user
/logout → logout user
/tasks → show/add/update/delete tasks
/calendar → calendar view
/notes → manage notes
/export/pdf → export tasks to PDF
/export/csv → export tasks to CSV

Use Flask Blueprints for modular routing (auth, tasks, main).

SQLite3 models:

Users(id, name, email, password_hash)

Tasks(id, user_id, name, description, date, time, frequency, status)

Notes(id, user_id, content, date_created)
